🎯Objetivos do Projeto

Este projeto foi desenvolvido como parte de uma atividade acadêmica com o objetivo de aplicar, na prática, os conceitos fundamentais de desenvolvimento web utilizando apenas HTML e CSS.

A proposta principal foi criar uma interface funcional, visualmente agradável e organizada, respeitando as limitações impostas pelo trabalho, especialmente a ausência de JavaScript.

Além disso, o projeto buscou reforçar habilidades como estruturação semântica, estilização avançada e construção de layouts responsivos.

👥 Público-Alvo

O sistema foi pensado para:

  • Usuários interessados em conteúdo relacionado a jogos
  • Avaliadores acadêmicos (professores)
  • Desenvolvedores iniciantes que desejam entender estruturas de layout
  • Usuários que acessam via desktop e dispositivos móveis

🛠 Tecnologias Utilizadas

Durante o desenvolvimento, foram utilizadas exclusivamente:

  • HTML5 → Para estruturação do conteúdo
  • CSS3 → Para estilização, layout e responsividade

Embora o uso de Bootstrap fosse permitido, optou-se por não utilizá-lo, priorizando a implementação manual para maior aprendizado e controle do código.

⚙ Restrições e Requisitos Mínimos

📌Restrições do projeto:

  • Não foi permitido o uso de JavaScript autoral
  • Interações limitadas apenas a HTML e CSS
  • Sem uso de bibliotecas externas além das permitidas

💻Requisitos mínimos para o usuário:

  • Navegador moderno (Chrome, Firefox, Edge)
  • Conexão com internet
  • Resolução mínima recomendada de 768px

🚧 Dificuldades

Durante o desenvolvimento, algumas dificuldades foram identificadas:

  • Criação de interatividade sem JavaScript
  • Diferenças de comportamento entre navegadores (principalmente Firefox)
  • Ajustes de alinhamento com Flexbox
  • Controle de responsividade sem frameworks prontos
  • Implementação de componentes mais complexos apenas com CSS

Esses desafios contribuíram significativamente para o aprendizado técnico.

✅ Pontos Fortes

  • Código totalmente manual (sem dependência de frameworks)
  • Maior controle sobre o design e layout
  • Boa organização estrutural
  • Interface limpa e funcional
  • Compatibilidade com navegadores modernos

❌ Pontos Fracos

  • Limitação de interatividade devido à ausência de JavaScript
  • Maior tempo de desenvolvimento
  • Necessidade de resolver problemas manualmente
  • Algumas limitações em recursos dinâmicos

🔄 Atualizações

Por se tratar de um projeto acadêmico, as atualizações são pontuais, ocorrendo principalmente para:

  • Correção de erros
  • Ajustes visuais
  • Melhorias na organização do código

Não há uma periodicidade fixa de manutenção.

🧩 Bootstrap

Apesar de o Bootstrap estar disponível como ferramenta permitida, ele não foi utilizado no desenvolvimento final.

A decisão de não utilizá-lo foi tomada com o objetivo de:

  • Praticar a construção de layouts do zero
  • Desenvolver maior domínio de CSS
  • Evitar dependência de frameworks
  • Compreender melhor o funcionamento de flex e responsividade

Dessa forma, todo o layout e estilização foram implementados manualmente.
